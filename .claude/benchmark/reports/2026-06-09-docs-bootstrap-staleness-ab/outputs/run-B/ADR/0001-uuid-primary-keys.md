# ADR 0001 — UUID как первичные ключи вместо autoincrement integer

**Дата решения:** 2024-07-19 (миграция `d98dd8ec85a3`)
**Статус:** accepted

## Context

Изначально модели (`User`, `Item`) использовали autoincrement-integer первичные
ключи. Это создаёт несколько проблем для API-first приложения:

- Последовательные числовые id предсказуемы и инкрементальны — позволяют
  enumeration ресурсов и утечку оценок объёма данных через URL/ответы API.
- Id присваивается только после INSERT, что усложняет сценарии, где ключ нужен
  до записи (клиентская генерация, идемпотентные операции, мерж данных из разных
  источников).
- Числовые id плохо мигрируют между средами/шардами (риск коллизий).

## Decision

Перейти на `uuid.UUID` первичные ключи во всех моделях, генерируемые на стороне
приложения: `id: uuid.UUID = Field(default_factory=uuid.uuid4, primary_key=True)`
(`backend/app/models.py`). Внешние ключи (`Item.owner_id`) тоже UUID. Миграция
`d98dd8ec85a3` выполняет полный перевод существующих таблиц: добавляет
`new_id`-колонки, заполняет их `uuid_generate_v4()`, переносит FK-связи и
переключает PK (требует расширение `uuid-ossp` в PostgreSQL).

## Consequences

**Плюсы:**
- Id больше не угадываются и не раскрывают объём/порядок данных.
- Ключ доступен до INSERT (`uuid4()` в Python), упрощает тесты и сидинг.
- Безопасный мерж/перенос данных между средами без коллизий.

**Минусы / издержки:**
- UUID занимает 16 байт против 4–8 у int; индексы крупнее, случайные UUIDv4
  хуже ложатся в B-tree-локальность, чем монотонные ключи.
- URL’ы и логи менее читабельны.
- Требуется PostgreSQL-расширение `uuid-ossp` (создаётся миграцией).

Эти издержки сочтены приемлемыми для шаблона, ориентированного на публичный API.

## Alternatives considered

- **Оставить autoincrement int** — отклонено из-за предсказуемости id и проблем
  переноса данных.
- **UUIDv7 / ULID (сортируемые по времени)** — лучше locality индекса, но на
  момент решения не было встроенной поддержки в stdlib (`uuid` в Python <3.11
  не имеет v7); добавило бы зависимость. Можно пересмотреть при апгрейде
  (UUIDv7 в `uuid` появился в Python 3.14).
- **BIGINT с обфускацией (hashids)** — прячет последовательность, но не решает
  pre-INSERT-генерацию и кросс-средовый мерж; добавляет слой кодирования.
