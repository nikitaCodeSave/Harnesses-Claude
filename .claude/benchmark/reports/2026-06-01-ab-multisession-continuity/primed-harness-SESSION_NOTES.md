## Ключевые доменные решения в промпте

1. **ОСТАТКИ (CA_LCY_SUM, CA_LCY_RUB_SUM, DEP_SUM)** → `AVG()` за период >1 мес, не SUM.

2. **ПОТОКИ (PNL_SUM, FX_*, PL_*, VED_*)** → `SUM()` всегда.

3. **Фильтр по времени**: `TRUNC(MONTH_DT, 'MM') = DATE 'YYYY-MM-01'` для одного месяца;
   `BETWEEN DATE 'YYYY-QQ-01' AND DATE 'YYYY-QQ-01'` для квартала.
   - MONTH_DT хранит EOM (конец месяца) для закрытых месяцев, поэтому прямое равенство
     по дате ненадёжно — нужен TRUNC.

4. **Подсчёт клиентов**: `COUNT(DISTINCT INN)` — один клиент = N строк (по числу месяцев).

5. **Oracle синтаксис**: `FETCH FIRST N ROWS ONLY` вместо LIMIT.

6. **NO_SQL**: если вопрос не из этой таблицы.

7. **Новые клиенты по месяцам**: фильтровать по `CREATE_DT`, не по `MONTH_DT`.
   Группировать: `GROUP BY TRUNC(CREATE_DT, 'MM')`.

8. **Сравнение периодов** (QoQ, YoY): использовать WITH CTE на каждый период, соединять
   декартовым произведением (`FROM cte1, cte2` — оба возвращают ровно 1 строку).
   Разница: `(p2 - p1)`. Процент: `ROUND((p2 - p1) / NULLIF(p1, 0) * 100, 2)`.

9. **Структура дохода**: SUM(PL_CA) + SUM(PL_DEP) + SUM(PL_VK) + SUM(PL_VK_OUT) +
   SUM(FX_MARGIN_RUB) + SUM(VED_SUM) + прочее.
   Прочее = SUM(PNL_SUM) - сумма всех перечисленных компонентов.

10. **Последний доступный месяц**: подзапрос
    `(SELECT MAX(TRUNC(MONTH_DT, 'MM')) FROM client_product)`.
