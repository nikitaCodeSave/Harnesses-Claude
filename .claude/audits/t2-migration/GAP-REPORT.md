# Claude Code harness audit — AI_analyst_migration

- **Дата:** 2026-06-10 · **Режим:** Audit (kit `claude-code-harness`, T2 gap-report)
- **Вход:** вердикты 5 независимых аудиторов (checklist §1–10, agents, skills, continuity, config), дедуплицированы synthesizer'ом
- **Скоуп:** `.claude/`, корневой `CLAUDE.md`, `devlog/`, `docs/` — read-only; в проект не внесено ни одной правки

## Summary

Зрелый, активно используемый harness 2026-Q1-эры (13 skills, 7 agents, 3 hooks, 7 rules; devlog жив до 2026-05-28, 138 записей; цепочка /think→/research→/spec→/implement→/verify→/review→/sync-docs→/compound реально wired). Три доминирующих класса дефектов: **(1) over-machinery**, кодирующая «модель не умеет X нативно» — обязательная 3-субагентная TDD-тройка per task, sync-docs agent+skill (ретайрнут в lab-прецеденте), parallel-execute поверх built-in dynamic workflows; **(2) wiring/staleness** — неподключённый ruff-hook, pipeline-knowledge skill, скармливающий 7 forked-агентам устаревшую архитектуру, противоречащую CLAUDE.md; **(3) нулевая permissions-поверхность** — ни одного deny при живых банковских данных (.env с кредами, allowed_users.yaml с реальными email). Continuity-слой Phase 5 kit'а отсутствует целиком (нет init.sh-оракула, features.json, .claude/progress/). Топ-3 гэпа: permissions-блок (G1), staleness pipeline-knowledge (G7), инверсия TDD-дефолта в /implement (G9).

## Findings

Дедуплицировано: 17 CHK + 7 agents + 13 skills + 6 CON + 6 CFG вердиктов → 24 уникальных gap'а. Колонка «Источники» хранит provenance объединённых находок; полный evidence — в JSON-вердиктах аудиторов (см. шапку).
